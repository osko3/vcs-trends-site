What is Version Control?

Version Control Systems (VCS) track changes to files and enable collaboration on software projects. They maintain history, support rollback, and allow multiple developers to work together efficiently.

Centralized vs. Distributed Models

  • Centralized VCS (CVCS): One central server stores the official code (e.g., CVS, Subversion).
  • Distributed VCS (DVCS): Every user has a full copy of the repository (e.g., Git, Mercurial).

Distributed systems revolutionized collaboration by enabling offline work, better branching and merging, and flexible workflows. Git, created by Linus Torvalds in 2005, is now the de facto standard.
